# D:\model_picture 金属零件缺陷分割训练操作手册

## 项目背景

本项目基于 **STM32MP157DAA1** 工业缺陷检测与分拣系统，目标是检测金属零件（垫圈、冲压件等）表面缺陷，包括划痕、锈蚀、压伤、裂纹、毛刺五类。最终模型将以 INT8 量化形式部署到 STM32MP157（双核 Cortex-A7 @ 800MHz，无 GPU）上运行。

**选型方案**：UNet + MobileNetV3-Small（方案B）
- UNet 的跳跃连接能保留细小缺陷的边界信息
- MobileNetV3-Small backbone 保证模型轻量，能在 MP157 上推理
- 预训练 ImageNet 权重加速收敛，小数据集也能训练

**本地训练环境**：
- GPU：NVIDIA GeForce RTX 4060 Laptop（8GB 显存）
- CUDA Driver：12.5（nvidia-smi 显示）
- PyTorch：2.6.0+cu124（CUDA Runtime 12.4，兼容 12.5 驱动）
- Python：3.9
- Conda 环境：`D:\model_picture\defect-unet`

## 3. 训练数据规范

### 3.1 类别定义

| 像素值 | 类别 | 英文标签 | 说明 |
|---|---|---|---|
| 0 | 背景 | background | 正常区域、无缺陷 |
| 1 | 划痕 | scratch | 表面线性划伤 |
| 2 | 锈蚀 | rust | 表面氧化锈斑 |
| 3 | 压伤 | dent | 凹陷、磕碰痕迹 |
| 4 | 裂纹 | crack | 细小裂纹 |
| 5 | 毛刺 | burr | 边缘冲压毛刺 |

### 3.2 数据拍摄要求

| 要求 | 说明 |
|---|---|
| 用实际工位的 UVC 摄像头拍 | 不要用手机，镜头畸变和色彩不同 |
| 用实际工位的光源 | 补光角度、色温必须一致 |
| 分辨率 640x480 | 你的UVC摄像头默认分辨率 |
| 零件居中、固定焦距 | 手动对焦后锁定 |
| 每张图只有一个零件 | 避免多零件干扰 |
| 不要后期调色 | 模型要学原始特征 |

### 3.3 数量要求

| 类别 | 最少 | 推荐 | 理想 |
|---|---|---|---|
| 良品 | 500张 | 1000张 | 2000+张 |
| 每类缺陷 | 150张 | 300-500张 | 500+张 |

### 3.4 原始数据存放

拍摄后的原始图片和 Labelme 标注 JSON 按缺陷类型分目录存放在 `raw_images` 下：

## 4. 数据标注流程（Labelme）

### 4.1 启动 Labelme

```powershell
D:\model_picture\defect-unet\python.exe -m labelme ./raw_images --labels scratch,rust,dent,crack,burr
```

### 4.2 标注操作

1. 在 Labelme 左侧选择一张图片
2. 右侧点击 "Create Polygons"（创建多边形）
3. 沿缺陷边缘点击画多边形，双击结束
4. 在弹出对话框中选择标签名（如 scratch）
5. 有多个缺陷就重复步骤 2-4
6. 良品图片直接保存（空标注 = 全背景）
7. File → Save 保存 JSON

### 4.3 标注质量要求

| 要求 | 说明 |
|---|---|
| 边界贴合缺陷 | 不要画太大也不要画太小 |
| 同一缺陷一个标注 | 不要把一个缺陷拆成多个多边形 |
| 良品也要保存 JSON | 空标注 = 全背景，防止误检 |
| 标签名必须严格匹配 | 只能用 scratch/rust/dent/crack/burr |

## 6. 训练操作

### 6.1 在 VSCode 中开始训练

1. 打开 VSCode
2. 文件 → 打开文件夹 → 选择 `D:\model_picture`
3. 按 `Ctrl + \`` 打开终端
4. 直接运行：

### 6.2 标准训练命令

```powershell
D:\model_picture\defect-unet\python.exe train.py --data_dir ./datasets --epochs 100 --batch_size 8
```

### 6.3 完整参数说明

| 参数 | 默认值 | 说明 |
|---|---|---|
| `--data_dir` | `./datasets` | 数据集根目录 |
| `--num_classes` | 6 | 类别数（背景+5种缺陷） |
| `--batch_size` | 8 | 批大小，显存不够改 4 或 2 |
| `--epochs` | 100 | 训练轮数，先跑 100 轮看效果 |
| `--lr` | 1e-3 | 学习率 |
| `--encoder` | mobilenet_v3_small | backbone 名称 |
| `--checkpoint_dir` | `./checkpoints` | 模型保存目录 |
| `--log_dir` | `./logs` | TensorBoard 日志目录 |
| `--num_workers` | 4 | 数据加载线程数 |

### 6.4 显存不够时调整

| 现象 | 解决 |
|---|---|
| CUDA Out of Memory | `--batch_size 4` |
| 4 也不够 | `--batch_size 2` |
| 还是不够 | 在 dataset.py 中把 Resize 改为 160x160 |

你的 RTX 4060 8GB 显存，batch_size=8 应该没问题。

### 6.5 用 TensorBoard 监控训练

另开一个终端：
```powershell
D:\model_picture\defect-unet\python.exe -m tensorboard.main --logdir D:\model_picture\logs
```

浏览器打开 **http://localhost:6006** 查看：
- loss 曲线（训练/验证）
- mIoU 曲线

## 7. 训练效果评估

### 7.1 核心指标：IoU（交并比）

IoU = 预测正确的缺陷像素 / (预测缺陷像素 + 真实缺陷像素)
mIoU = 所有类别 IoU 的平均值

**关键：不要只看 mIoU，背景 IoU 会拉高整体值，重点看缺陷类别的 IoU。**

### 7.2 效果等级

| 指标 | 可用 | 较好 | 很好 |
|---|---|---|---|
| mIoU | > 0.45 | > 0.60 | > 0.70 |
| 划痕 IoU | > 0.40 | > 0.55 | > 0.65 |
| 锈蚀 IoU | > 0.50 | > 0.65 | > 0.75 |
| 裂纹 IoU | > 0.30 | > 0.45 | > 0.60 |
| 压伤 IoU | > 0.35 | > 0.50 | > 0.65 |
| 毛刺 IoU | > 0.40 | > 0.55 | > 0.70 |

## 8. 效果不好时的调整方法

### 8.1 常见问题速查

| 问题 | 原因 | 解决方法 |
|---|---|---|
| 损失不下降 | 学习率太大或太小 | `--lr 5e-4` 或 `--lr 2e-3` |
| 损失剧烈震荡 | 学习率太大 | `--lr 5e-4 --batch_size 4` |
| 训练好但验证差 | 过拟合 | 增加数据量、增加数据增强 |
| 某类缺陷 IoU=0 | 该类样本太少 | 专门补充该类样本 200+张 |
| 掩码全黑/全 0 | 掩码格式错误 | 用图像查看器检查掩码 PNG 像素值 |
| 掩码边界粗糙 | 分割头不够精细 | 换 encoder_depth=5（改 train.py） |
| 整体 IoU 偏低 | 数据量不够 | 每类至少 300 张 |

### 8.2 分步调优策略

```
训练结果不理想？
  │
  ├── 1. 先看数据
  │     ├── 某类样本 < 150张 → 补充该类样本
  │     ├── 良品样本 < 500张 → 补充良品样本
  │     └── 掩码标注不准 → 重新标注边界
  │
  ├── 2. 再调参数
  │     ├── lr 从 1e-3 → 5e-4
  │     ├── epochs 100 → 150（多训几轮）
  │     └── batch_size 8 → 4（小 batch 有时更好）
  │
  ├── 3. 改损失函数
  │     ├── 缺陷面积小 → 增大 Dice 权重（改 train.py 中 0.5 → 1.0）
  │     └── 类别极度不平衡 → 加 Focal Loss
  │
  ├── 4. 换模型
  │     ├── mobilenet_v3_small → mobilenet_v3_large（更强但更大）
  │     └── smp.Unet → smp.UnetPlusPlus（更精细的解码器）
  │
  └── 5. 仍不达标
        └── 考虑方案A（MobileNetV3+LR-ASPP）
            或增加注意力机制
```

### 8.3 从检查点恢复训练

如果训练中断，可以从保存的检查点恢复：

```powershell
# 从第 20 轮的检查点恢复（需修改 train.py 添加恢复逻辑）
```

## 12. 常见问题排查

| 问题 | 原因 | 解决 |
|---|---|---|
| `conda` 命令找不到 | PowerShell不支持conda_hook.bat | 直接用 `D:\model_picture\defect-unet\python.exe` 运行脚本 |
| `torch.cuda.is_available()` 返回 False | CUDA 版本不匹配 | 确认安装了 cu124 版 PyTorch |
| `No module named smp` | 用了系统Python而非conda环境 | 用完整路径 `D:\model_picture\defect-unet\python.exe` |
| `cv2.imshow` 报错 "function not implemented" | 装了opencv-python-headless（无GUI版） | `pip uninstall opencv-python-headless`，重装 `opencv-python` |
| CUDA Out of Memory | batch_size 太大 | 改 `--batch_size 4` |
| Loss=NaN | 学习率太大 | 改 `--lr 1e-4` |
| 图片和掩码数量不一致 | 转换时部分失败 | 检查 raw_images 中的 JSON 是否完整 |
| Labelme 中文乱码 | 编码问题 | 用 `--labels` 参数指定英文标签 |
| BrokenPipeError | Windows多进程问题 | 改 `--num_workers 0` |
